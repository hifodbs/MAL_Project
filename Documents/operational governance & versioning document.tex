% 2. Version Control Strategy
% =================================================
\section{Version Control Strategy}
A GitFlow branching model is adopted.
Code, datasets, and models are versioned using semantic versioning.

% =================================================
% 3. CI/CD \& Automation
% =================================================
\section{CI/CD \& Automation}
\begin{itemize}
    \item Run tests
    \item Validate data
    \item Train and evaluate models
    \item Deploy models if KPIs are satisfied
\end{itemize}

% =================================================
% 4. Model Lifecycle Governance
% =================================================
\section{Model Lifecycle Governance}
Models are registered in a centralized registry with metadata including:
\begin{itemize}
    \item Hyperparameters
    \item Training dataset version
    \item Evaluation metrics
\end{itemize}

% =================================================
% 5. Monitoring \& Maintenance Plan
% =================================================
\subsection{Monitoring and Maintenance}

\subsubsection{Monitoring}
Feature drift and prediction performance are continuously monitored.

\subsubsection{Incident Response}
If degradation is detected, the system triggers retraining or rolls back to a previous stable model.
